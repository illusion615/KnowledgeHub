01 / Pricing Framework

先把报价对象拆成 3 类消耗

Power Platform 报价最容易出错的地方，不是数字本身，而是把不同计费对象混在一起。实践中建议统一拆分为：用户授权、容量包、使用量计费。

用户授权

谁在用

例如 Power Apps Premium、Power Automate Premium、Power BI Pro。适合已知固定人群、长期使用的场景。

容量包

需要多少吞吐

例如 Power Pages 的 Authenticated / Anonymous 容量包，Power Automate Process 按 bot 计费。适合大流量、共享型服务。

使用量计费

用了多少算多少

例如 Copilot Studio 的消息或 Copilot Credits、Power Apps 的 Pay-as-you-go。适合需求不确定或先试点再放大。

02 / Licensing Rules

产品级报价规则速览（用于建模）

下列规则提炼自官方 licensing FAQ 与产品文档。实际采购以你的地区、协议类型和当期目录价为准。

主规则

Power Apps Premium 适合多应用、持续使用的人群。Per app 适合单场景起步，按用户与应用组合计价。

参考价（USD）

Premium：$20/用户/月；大批量新增用户可能有阶梯价（文档示例提到 2,000+ 新增用户可到 $12）。Per app：$5/用户/应用/月。

建模建议

同一批用户平均使用应用数 ≥2 时，优先评估 Premium；平均 <2 且范围明确时，per app 往往更稳妥。

主规则

有人值守与云流自动化通常按 Premium 用户授权；无人值守 RPA、共享流程按 Process（bot）扩展。

参考价（USD）

Premium：$15/用户/月；Process：$150/bot/月；Hosted RPA add-on：$215/bot/月。

建模建议

把“人触发”与“机触发”流程分开估算，避免把高频无人值守负载塞进按人授权导致预算偏差。

主规则

Authenticated 与 Anonymous 分开买容量包。Authenticated 按 100 用户/包，Anonymous 按 500 用户/包。

参考价（USD）

Authenticated：Tier1 $200/包，Tier2 $75/包（100 包起），Tier3 $50/包（1,000 包起）。Anonymous：Tier1 $75/包，Tier2 $37.5/包（20 包起），Tier3 $25/包（200 包起）。

建模建议

必须比较跨 tier 的总成本，不要只看单包单价。高 tier 通常有最低采购门槛，低流量时可能更贵。

Copilot Studio 规则

Tenant 级预付容量包含每月 25,000 消息；常规消息按 1、生成式回答按 2 计。2025 年后官方统一口径为 Copilot Credits，但计量思想保持一致。

Power BI 规则

在共享容量中，协作发布通常要求 Pro/PPU；在 Premium 容量可放宽消费端授权。价格请以 Power BI pricing 页面实时值为准。

建模建议

Copilot 先按消息量估预算，再做对话设计优化；Power BI 先确认是否需要“广泛分发”再决定按人还是容量。

主规则

Dataverse 存储分 Database、File、Log 三类，各用户许可按人数累加到租户池。超出部分按 GB 购买 add-on。默认环境额外赠送 3 GB Database + 3 GB File + 1 GB Log。

参考价（USD）

Database：$40/GB/月；File：$2/GB/月；Log：$10/GB/月。

每用户附带容量

Power Apps Premium：+250 MB DB / +2 GB File；Per App：+50 MB DB / +400 MB File。Power Automate Premium：+250 MB DB / +2 GB File。Copilot Studio 租户级：5 GB DB / 20 GB File / 2 GB Log。

建模建议

先估算许可附带总容量（用户数 × 每用户容量 + 默认赠送），再与实际业务数据量对比，差额部分才需要购买 add-on。File 容量通常是附件驱动的，Database 容量由记录行数驱动。

04 / Business Scenario Translator

业务场景 → AI 产品配置：用业务语言估算

选择一个业务场景模板，填入你的业务量（如每月订单数、每份文档页数），系统自动将其转换为 Copilot Credits 需求并合并到总报价中。

📄 发票 / 收据识别

每月处理多少张发票或收据？每张多少页？系统按 8 Credits/页计算。

📝 自定义文档处理

订单表单、合同、申请单等自定义模板识别。按 8 Credits/页（自定义模型）。

🔍 文字识别 OCR

批量扫描纸质文档提取文字。按 0.1 Credits/页（极低成本）。

🤖 智能客服 Agent

客户问答机器人，混合经典回答和生成式回答。按对话次数和每次回答模式估算。

✨ AI Prompt 调用

流程中调用 GPT Prompt（摘要、分类、翻译等）。按调用次数和平均 token 数估算。

    06 / Configuration Playbook

    从试点到规模化的配置路径

    报价不是一次性动作，建议按阶段滚动：先用最小可行组合验证业务价值，再沿着使用量增长曲线替换授权模型。

    1. 1
      试点期（1-2 个场景）

      优先 per app + 少量 Premium 用户 + Pages Tier1，关注真实活跃用户和自动化触发次数。

    2. 2
      放大期（跨部门推广）

      把高复用人群切到 per user；无人值守流程独立购买 Process；门户按月峰值重算容量包。

    3. 3
      治理期（预算与性能联动）

      把 Copilot 消息预算、Power Platform Requests、Dataverse 容量拉进同一看板，用阈值预警驱动采购节奏。
