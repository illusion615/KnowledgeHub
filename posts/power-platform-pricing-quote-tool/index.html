01 / Pricing Framework

Power Platform 的 3 种计费模式

理解报价的第一步是区分三种计费模式。企业采购通常使用「用户授权」和「容量包」（预付费），而 Pay-as-you-go 适合试点或弹性场景。

用户授权 (Per User)

按人头付费，预付年包

为每位使用者购买许可，按年预付、按月计价。适合已知固定人群、长期使用的场景。

  • Power Apps Premium — $20/user/mo
  • Power Automate Premium — $15/user/mo
  • Power BI Pro — $10/user/mo
  • 每位用户附带 Dataverse 容量
容量包 (Capacity Pack)

按吞吐量预购，共享使用

按容量单位（用户包 / Credits 包 / Bot）预购，不绑定特定用户，适合大流量和 AI 场景。

  • Power Pages — $200/100 人包/mo
  • Copilot Studio — $200/25k Credits/mo
  • AI Builder — $500/1M Credits/mo
  • Power Automate Process — $150/bot/mo
Pay-as-you-go

先用后付，Azure 按量结算

通过 Azure 订阅按实际用量月结，无预付承诺。适合试点验证、弹性需求或无法预估用量的场景。

  • Power Apps — $10/active user/app/mo
  • Power Automate — $0.60/cloud flow run
  • Power Pages — $4/auth user/mo
  • Copilot Credit — $0.01/credit

业务场景 Token 折算参考

GPT 系列模型中，1 token ≈ 0.7 个英文单词 ≈ 0.5 个中文字符。下表为常见处理对象的平均 token 消耗（含 Prompt 指令 + 内容 + 输出，按单次调用估算）：

处理对象中文（tokens）英文（tokens）说明
一封邮件摘要/分类0.8k – 1.5k0.5k – 1k含邮件正文 + 分类/摘要指令 + 输出
一页合同/文档摘要1.5k – 2.5k0.8k – 1.5kA4 密排约 800 中文字 ≈ 1.6k tokens
一条客服对话（单轮）0.3k – 0.8k0.2k – 0.5k用户提问 + 系统 Prompt + 回答
一份表单字段提取0.5k – 1k0.3k – 0.8k结构化提取 5–15 个字段
一段文本翻译（500 字）1.5k – 2k1k – 1.5k原文 + 译文双向消耗
一篇报告生成（1000 字）2.5k – 4k1.5k – 2.5k含大纲指令 + 全文输出

示例：每月处理 2,000 封中文邮件做摘要分类，按均值 1.2k tokens/封 × 标准模型费率 1.5 Credits/1k = 每封 1.8 Credits × 2,000 = 3,600 Copilot Credits/月。在上方试算表「Prompt（标准模型）」行填入 2,400（= 2,000 × 1.2）即可。

组合计费示例

客服 Agent 每次对话平均 4 次经典回答 + 2 次生成式回答 = (4×1)+(2×2) = 8 Credits/次。900 客户/天 = 7,200 Credits/天 × 30 天 = 216,000 Credits/月。

    05 / Configuration Playbook

    从试点到规模化的配置路径

    报价不是一次性动作，建议按阶段滚动：先用最小可行组合验证业务价值，再沿着使用量增长曲线替换授权模型。

    1. 1
      试点期（1-2 个场景）

      以最小用户数购买 Premium 许可 + Pages Tier1，关注真实活跃用户和自动化触发次数。

    2. 2
      放大期（跨部门推广）

      按实际活跃用户扩展 Premium 许可；无人值守流程独立购买 Process；门户按月峰值重算容量包。

    3. 3
      治理期（预算与性能联动）

      把 Copilot Credits 预算、Power Platform Requests、Dataverse 容量拉进同一看板，用阈值预警驱动采购节奏。
