01 / Model Overview

模型概览与核心特性

理解 Z-Image-Turbo 的架构和能力边界，是写好提示词的前提。

Z-Image-Turbo 采用 S3-DiT（Scalable Single-Stream Diffusion Transformer）架构，将文本嵌入、视觉语义 token 与图像 VAE token 拼接为单一序列，最大化参数利用率。

核心加速来自 Decoupled-DMD 蒸馏算法，将 CFG Augmentation（驱动器）和 Distribution Matching（正则器）解耦优化，最终实现仅 8 步推理即达到 50 步模型的品质。后续 DMDR 进一步融合强化学习，提升语义对齐与高频细节。

对提示词写作的直接影响：模型已内置较强的语义理解能力，提示词无需像 SD/SDXL 那样依赖精细的权重标签堆叠，而应侧重自然语言描述 + 结构化分层。

📸 摄影级真实感

精细控制细节、光影和纹理，兼顾高保真与构图美学。适合商业摄影、产品图、人像写真。

✍️ 中英文渲染

开源模型文字渲染 SOTA，支持多行布局、段落级文本、小字号场景，海报设计表现突出。

02 / Prompt Formula

提示词公式

根据阿里云百炼官方文档，提示词描述越完整、精确和丰富，生成图像品质越高。以下两层公式适用于 Z-Image-Turbo。

prompt = 主体 + 场景 + 风格
  • 主体：图片的主要表现对象 — 人、动物、植物、物品或想象之物
  • 场景：主体所处环境 — 室内/室外、季节、天气、光线、真实或虚构空间
  • 风格：艺术风格 — 写实、3D 卡通、水彩、水墨、折纸等

示例：人像摄影

Z-IMAGE-TURBO
25 岁中国女孩，圆脸，看着镜头，优雅的民族服装，商业摄影，室外，电影级光照，半身特写，精致的淡妆，锐利的边缘。
适用场景：初次使用 AI 创作、灵感探索阶段。简单自由的提示词反而能生成更具想象力的图像。
prompt = 主体描述 + 场景描述 + 风格定义 + 镜头语言 + 氛围词 + 细节修饰
  • 主体描述：包含特征、动作、表情 — "穿着红色外套的 10 岁中国小女孩"
  • 场景描述：环境细节用形容词或短句列举 — "晨雾中的竹林小径"
  • 风格定义：明确艺术风格/表现手法 — "吉卜力工作室风格水彩"
  • 镜头语言：景别 + 视角 + 镜头类型 — "微距特写，俯视视角"
  • 氛围词：情绪/感受 — "温馨、怀旧、梦幻"
  • 细节修饰：材质/质量增强 — "8K，超高清，精致的纹理"

示例：羊毛毡 3D 角色

Z-IMAGE-TURBO
由羊毛毡制成的大熊猫，头戴大檐帽，穿着蓝色警服马甲，扎着腰带，携带警械装备，戴着蓝色手套，穿着皮鞋，大步奔跑姿态，毛毡效果，周围是动物王国城市街道商户，高级滤镜，路灯，动物王国，奇妙童趣，憨态可掬，夜晚，明亮，自然，可爱，4K，毛毡材质，摄影镜头，居中构图，毛茸茸。

04 / Advanced Techniques

进阶提示词策略

从官方技术博客和社区实践中提炼的高阶技巧。

Z-Image-Turbo 在阿里云百炼 API 中提供 prompt_extend 参数（默认开启），它会使用大语言模型将简短提示词扩写为结构化的详细描述。

开启 prompt_extend 时：模型会返回优化后的提示词及推理过程。适合创意探索阶段 — 输入简短想法，让 PE 补全细节。
关闭 prompt_extend 时：适合对画面细节要求精确可控的场景。此时需要自行提供完整的进阶公式描述。

PE 推理示例：古诗可视化

输入：

帮我给《登科后》配图，最出名的两句

PE 会自动推理出"春风得意马蹄疾，一日看尽长安花"，并生成包含策马长安、花团锦簇的详细场景描述。

PE 推理示例：逻辑题可视化

输入：

鸡兔同笼，共有 35 个头和 94 只脚，求鸡和兔各有多少只

PE 先解方程得出 23 只鸡、12 只兔，再生成相应数量动物的画面。这是 Z-Image 区别于传统扩散模型的独特能力。

Z-Image-Turbo 对摄影术语的理解远超一般扩散模型。从官方示例可以看到，模型支持精细的胶片模拟参数。

官方示例：胶片人像

Z-IMAGE-TURBO
film grain, analog film texture, soft film lighting, Kodak Portra 400 style, cinematic grainy texture, photorealistic details, subtle noise, (film grain:1.2)。采用近景特写镜头拍摄的东亚年轻女性，呈现户外雪地场景...

关键观察：该模型支持类 SD 的权重语法 (关键词:权重)，如 (film grain:1.2)。虽然不如 SD 对权重的依赖程度高，但在需要强调特定视觉元素时仍然有效。

高效的摄影风格关键词组合：

  • 胶片感：film grain, analog film texture, Kodak Portra 400/Fuji Superia
  • 电影感：cinematic lighting, anamorphic lens, shallow depth of field
  • 商业摄影：studio lighting, product photography, clean background
  • 纪实风：documentary style, candid, natural light, 35mm

Z-Image-Turbo 原生支持中英文双语输入，这是其相比 FLUX、SD 等模型的显著优势。社区实测发现以下混合策略效果最佳：

  1. 场景与主体用中文：中文描述更精确地传达中国文化语境 — "青灰瓦顶、朱红色外墙的中式商铺"
  2. 风格与技术参数用英文：摄影/艺术术语英文更精确 — "cinematic lighting, 8K, Kodak Portra 400"
  3. 文字渲染内容用目标语言：如需渲染中文字，直接在提示词中写中文引号包裹的文字内容

示例：中英混合海报

Z-IMAGE-TURBO
冬日北京的都市街景，青灰瓦顶、朱红色外墙的两间相邻中式商铺比肩而立，檐下悬挂印有剪纸马的暖光灯笼，在阴天漫射光中投下柔和光晕。左侧为书法店：靛蓝色老旧的牌匾上以遒劲行书刻着"文字渲染"。cinematic lighting, photorealistic, 8K, fine details.

文字渲染是 Z-Image-Turbo 的标志性能力。以下技巧可提高渲染成功率：

  1. 用引号包裹目标文字：明确用引号标注需渲染的文字 — 牌匾上刻着"文字渲染"
  2. 指定载体：说明文字出现在什么载体上 — 牌匾、海报、T恤、霓虹灯牌
  3. 指定字体风格：行书、楷体、手写体、衬线体
  4. 控制文字数量：单次渲染建议不超过 10-15 个中文字符，过长的文本拆分为多行或多区域
  5. 开启 prompt_extend：PE 可自动补全文字排版细节，推荐在文字渲染场景开启

示例：对联渲染

Z-IMAGE-TURBO
一副典雅庄重的对联悬挂于厅堂之中，房间是安静古典的中式布置，桌子上放着青花瓷，对联上左书"义本生知人机同道善思新"，右书"通云赋智乾坤启数高志远"，横批"智启千问"，字体飘逸，在中间挂着一幅中国风的画作，内容是岳阳楼。

05 / Prompt Adherence

提示词遵从性：从评测数据到行动清单

Z-Image 论文 (arXiv:2511.22699) 公布了 7 项基准评测数据。以下将每项数据转化为具体可落地的提示词写法。

Z-Image 团队在 RLHF 阶段将"指令遵从"分解为 5 个可标注维度：(i) 核心主体实体、(ii) 属性规格、(iii) 动作/交互、(iv) 空间/构图约束、(v) 风格/渲染条件。人工标注员逐维点选不满足项，计算满足比率作为奖励信号。

这 5 个维度直接对应你写提示词时应覆盖的 5 个检查点。

评测基准 测量维度 Z-Image-Turbo 对比参考
GenEval 物体数量 / 颜色 / 位置 / 属性 Overall 0.82 GPT Image-1: 0.84
DPG-Bench 密集提示词遵从 Overall 85.92 Qwen-Image: 88.32
TIIF 指令遵从（基础/高级/设计师） 排名第 5 GPT Image-1: 排名第 1
OneIG-EN Text 维度 0.994 GPT Image-1: 0.857
GenEval Position（空间关系） 0.65 GPT Image-1: 0.75
AI Arena (Elo) 人类偏好（含 prompt alignment） 全球第 4 / 开源第 1 Imagen 4: 第 1
关键发现：文字渲染（0.994）和属性描述（91.59）是绝对强项；空间位置关系（0.65）是相对短板。下面的行动清单就是围绕"扬长补短"展开。

基于 RLHF 奖励模型的 5 维拆解，每次写完提示词后用以下清单逐条核对：

维度 需要写什么 示例 缺失后果
1. 核心主体 明确的主体名词 + 数量 "一只橘猫""两个小女孩" 主体缺失或数量错误
2. 属性规格 颜色、材质、服装、年龄 "穿着红色棉麻连衣裙的 25 岁女性" 颜色/服装随机
3. 动作/交互 主体的动作 + 主体间关系 "猫蹲在女孩肩膀上" 姿态默认站立
4. 空间/构图 前后左右上下的位置关系 "女孩在画面左侧，背景右侧是一棵樱花树" 布局随机（模型短板）
5. 风格/渲染 艺术风格 + 光线 + 镜头 "日系胶片风，逆光，85mm 长焦" 默认写实风格

实操演示：五维一步到位

Z-IMAGE-TURBO · PE OFF
【主体】一只橘色虎斑猫蹲在【动作】一位 25 岁亚洲女性的左肩上，女性【属性】穿着白色亚麻衬衫、戴圆形金属眼镜，【空间】站在画面中央偏左，背景右侧是一棵盛开的粉色樱花树，前景散落着花瓣。【风格】Fuji Superia 400 胶片风格，逆光，golden hour，85mm 长焦浅景深，暖色调。

GenEval Position 维度得分 0.65 是模型的已知短板。以下 3 种策略可显著提升空间遵从率：

  1. 用"画面坐标"替代模糊方位词

    避免 → "猫在女孩旁边"
    改为 → "猫在画面左侧 1/3 处，女孩在右侧 2/3 处"

  2. 使用摄影构图术语

    模型对专业构图词的遵从度高于日常描述：
    三分法构图 / 居中对称构图 / 对角线构图 / 框架式构图

  3. 开启 prompt_extend 处理空间任务

    PE 推理链会将模糊的空间意图扩展为精确的场景布局描述，在空间关系任务上比手写效果更稳定。

对比：空间关系弱描述 vs 强描述

弱（可能失败）
一只猫和一条狗在公园里
强（遵从率高）
三分法构图，画面左侧 1/3 处是一只橘猫正面端坐于草地上，右侧 2/3 处是一只金毛犬侧身奔跑，中景是公园长椅和散步的行人，背景是模糊的梧桐树林，自然光，平视视角。

OneIG Text 维度 0.994 是所有评测模型中最高分。以下是从论文和官方示例中提炼的渲染成功率最高的写法模式：

文字渲染模板 = "载体" + 上/中/下位置 + 字体风格 + "引号包裹的渲染内容"

模式 A：载体 + 位置 + 内容（最高遵从率）

Z-IMAGE-TURBO
一块靛蓝色老旧的牌匾，上面以遒劲行书刻着"文字渲染"

模式 B：海报中多区域文字（论文 Appendix A.2 模式）

Z-IMAGE-TURBO
竖版海报设计，深蓝色背景。顶部是大号黄色字体主标题"夢中天堂"，主标题下方是较小的白色无衬线字体"HEAVEN OF DREAM"。底部用较小字体列出日期"2025.11.11"和地点"阿里巴巴云谷园区"。
经验法则：单次渲染 ≤ 15 个中文字 / 30 个英文字母时准确率最高。超长文本应拆分为多个位置区域（"顶部标题""底部署名"）分别描述。

prompt_extend 不是"默认全开"这么简单。论文揭示 PE 的推理链在不同场景下效果差异大：

场景 PE 建议 原因
简短创意描述（< 30 字） 开启 PE 注入推理链补全场景细节和世界知识
需要推理的语义（古诗、数学题、坐标） 开启 PE 先解题再生成，模型本身无此推理能力
精确空间构图 + 多物体位置 开启 PE 扩展空间描述可补偿模型的 Position 弱项
已写好 100+ 字详细描述 关闭 PE 可能改写你精心设计的构图意图
精确文字渲染（已按渲染模板写好） 关闭 PE 改写可能丢失引号内精确文字
要求精确复现特定胶片/相机参数 关闭 PE 会添加它认为"更好"的描述覆盖你的参数

GenEval Attribute Binding 得分 0.68，意味着多物体 + 多属性时容易出现"属性漂移"（如红衣服跑到错误角色上）。解法：

  1. 紧密绑定法：属性紧贴主体名词

    避免 → "一个女孩和一个男孩，穿着红色和蓝色的衣服"
    改为 → "穿着红色连衣裙的女孩""穿着蓝色西装的男孩"

  2. 编号锚定法：为多主体编号

    "人物 1：穿白色衬衫的亚洲女性，站在左侧。人物 2：穿黑色夹克的欧洲男性，坐在右侧。"

  3. 减少同类物体数量

    需要 3 个以上同类物体（如 3 只不同颜色的猫），建议拆成多次生成 + 后期合成。模型在 2 个同类物体以内的属性绑定表现更稳定。

推荐分辨率列表（总像素约 1024*1024）：

  • 1:1 → 1024*1024
  • 2:3 → 832*1248
  • 3:2 → 1248*832
  • 9:16 → 720*1280
  • 16:9 → 1280*720

推荐分辨率列表（总像素约 1536*1536）：

  • 16:9 → 2688*1536
  • 9:16 → 1536*2688
  • 1:1 → 2048*2048
性能提示：推荐分辨率在 1024*1024 至 1536*1536 之间的出图效果最佳。超出此范围仍可生成图像但品质可能下降。
注意：Z-Image-Turbo 不支持 negative_prompt 参数。这与千问-文生图（Qwen-Image）和万相系列不同。如需排除特定元素，应在正向提示词中使用否定描述（如"无水印、无人物"），或依赖 prompt_extend 自动优化。

07 / Anti-Patterns

常见误区与解决方案

社区高频踩坑点与对应修正策略。

误区：将 SD/SDXL 的 tag 堆叠习惯直接搬到 Z-Image-Turbo — "masterpiece, best quality, ultra detailed, 8K, HDR, highres..."

原因：Z-Image-Turbo 基于强大的语言模型理解提示词，标签堆叠不仅冗余，还可能分散模型对核心语义的注意力。

修正：用自然语言完整描述场景，保留必要的品质修饰词（如 8K、超高清），去除通用模板词。

误区：只描述主体，不提供场景上下文与空间关系。

修正：按前景 → 中景 → 背景分层描述。例如："前景是一杯冒着热气的拿铁，中景是靠窗阅读的年轻女性，背景是雨后的城市街道，窗户上有雨滴。"

误区：文字内容写在描述中间，没有引号包裹，且单次渲染过多文字。

修正：(1) 用引号明确标注渲染文字；(2) 指定文字所在的载体/位置；(3) 单次渲染控制在 10-15 字以内；(4) 开启 prompt_extend 让 PE 自动补全排版细节。

误区 1：已经写了非常详细的提示词，还开着 prompt_extend — PE 的改写可能偏离你精心设计的构图意图。

误区 2：提示词极其简短（如"画只猫"），却关闭了 prompt_extend — 模型缺乏足够信息生成高质量图像。

原则：提示词越简短，越应开启 PE；提示词越详细且有精确构图要求，越应关闭 PE。
