03 / Dimension Scores

3. 九维度评分

A=优秀 B=良好 C=明显问题 D=严重风险。可测试性评 D 但本项目为纯静态站、不跑 CI，不触发红线停工。

# 维度 上次 本次 核心证据
1架构一致性ACarticle-presentation.js 混合 5 类职责（导出、讲述、录屏、设置面板、演示控制），无清晰模块分层。
2代码重复度ACShare dropdown 5 种模式各带独立事件处理器；CSS .home-link 规则出现两次；llm-literary-creation 内联 .quote-board 与共享 .quote-block 重复。
3命名与可读性AB函数命名清晰自解释；全局状态变量缺乏注释。
4错误处理AC多个异步操作 .catch() 静默失败；全屏拒绝、媒体权限失败无结构化恢复。
5可测试性AD全局状态紧耦合 DOM；无依赖注入；无法脱离完整页面做单元测试。
6性能与资源AB重模块懒加载（PptxGenJS、narration）；localStorage 无配额错误处理。
7安全AB无 XSS 向量；URL 生成安全；建议增加 CSP。
8设计语言复用BB色彩 token 统一；暗色模式规则可合并减少 30%；卡片系缺 .card-base。
9文档健康度BC演示模块无 README；JSDoc 覆盖率约 5%；架构决策未记录。
降档原因分析 上次审查时 article-presentation.js 仅 1,314 行且职责单一（演示步骤收集 + 导航 + PPT 导出调度），article-common.js 刚提取完成，架构处于最优状态。此后新增讲述系统（FAB + 设置面板 + 状态机 ~520 行）、录屏（161 行）、Email 导出（106 行），使文件膨胀到 2,158 行且职责混合。这不是代码质量退化，而是功能密集新增期的正常技术债累积。

# 维度 审查时 整改后 变化依据
1架构一致性CBJS 拆分为主控 + 3 懒加载模块；CSS 拆分为核心 + narration + diagram。职责边界清晰。仍有 ensureShareDropdown 245 行未拆，降为 B 而非 A。
2代码重复度CACSS .home-link 重复消除；全站 41 篇脚本统一；email/recording/narration-ui 各只实现一次。quote-board 仍为单篇独有，不构成重复。
3命名与可读性BA63 行架构注释覆盖 10 个功能模块 + 6 个懒加载接口 + 状态变量说明。
4错误处理CC未改。异步 .catch() 仍静默失败。后续可加 window.reportError。
5可测试性DCtests/validate.js 覆盖 7 项自动检查。但对 JS 行为仍无单元测试；纯静态站限制了测试深度。D→C。
6性能与资源BB未改。懒加载策略良好。
7安全BB未改。
8设计语言复用BACSS 拆分后 article.css 降到 2,398 行，远低于 3,000 预警线；narration/diagram 各自独立文件职责单一。
9文档健康度CB架构注释到位；article-pages.instructions.md 更新了 CSS/测试清单；repo memory 同步。
总结 9 项维度中 4 项提升（架构 C→B、重复 C→A、命名 B→A、设计语言 B→A）、2 项改善（可测试性 D→C、文档 C→B）、3 项持平（错误 C、性能 B、安全 B）。综合从 C+ 提升至 B+。剩余 1 项 P2 延期（quote-board 单篇使用不满足提取条件）合理。下一轮重点关注错误处理改善。
