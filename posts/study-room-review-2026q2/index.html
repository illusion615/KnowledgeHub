02 / Dimension Scores

2. 九维度评分

每项评 A/B/C/D 四档。A=优秀无需改动，B=良好有小改善空间，C=存在明显问题（2 轮内修复），D=严重风险（立即修复）。

维度评分评估依据
架构一致性B16/18 篇文章遵循统一骨架。2 篇（copilot-harness-engineering、enterprise-agent-architecture）使用 const/let + 箭头函数，违反 IE 兼容约定。共享 CSS/JS 分层清晰。
代码重复度BIntersectionObserver、accordion toggle、localStorage sync 在每篇文章中重复。应提取为共享 article-common.js。但组件级 CSS 复用度高（insight-grid 等被广泛使用）。
命名与可读性ACSS 类名语义清晰（.insight-card, .comparison-grid, .section-kicker）。JS 函数名自解释（collectPresentationSteps, syncContainerVisibility）。knowledge-data.js 字段命名一致。
错误处理AJS 中 null 检查充分（if (!site || !topbar || !hero || !main) return）。PPT 导出有 try/catch + 用户 alert。所有外部链接有 rel="noopener noreferrer"。
可测试性A通过浏览器 DevTools 可完整验证：暗色模式切换、演示模式步骤导航、PPT 导出、首页三种布局切换。无需额外测试框架。
性能AIntersectionObserver 使用 unobserve 释放已激活元素。Google Fonts 使用 preconnect。JS 文件懒加载（pptxgen.bundle.js 仅导出时加载）。无不必要的计算。
安全A所有 18 篇文章的外部链接均通过 target="_blank" + rel="noopener noreferrer" 审计（0 个遗漏）。无用户输入处理场景。
设计语言Carticle.css 2,828 行逼近膨胀红线。演示模式 CSS 规则复杂度高（:has() 选择器、多层覆盖）。部分组件（verdict-box、timeline-card、attr-card）仅在单篇文章的 inline style 中定义，未提取到共享 CSS。
文档健康度C双语覆盖严重不足：16/18 篇文章仅首页链接有 data-zh/data-en，主题内容全部为纯中文。knowledge-data.js 注册完整（18/18），但文章页 brand 文字与父主题一致性未强制验证。

03 / Key Findings

3. 关键发现

按严重程度排列。C 级项必须在下次审查前修复；B 级项择机优化。

C-1: 双语覆盖严重不足

16/18 篇文章仅有 home-link 的 data-zh/data-en 属性（值均为"←"）。文章主体、section 标题、hero 内容全部为纯中文硬编码。copilot-instructions.md 要求"所有文本元素支持 zh/en"，但实际执行率接近 0%。仅 study-room-design（180 个属性）和 claw-code-analysis（150 个属性）有实质双语内容。

C-2: 文章内 JS 模板代码重复

每篇文章底部的 IntersectionObserver 初始化（~15 行）、accordion toggle 绑定（~25 行）、localStorage 主题/语言同步（~12 行）完全相同，在 18 篇文章中重复 18 次，总计约 936 行重复代码。应提取为共享 article-common.js。

B-1: 2 篇文章使用 const/let + 箭头函数

copilot-harness-engineering 和 enterprise-agent-architecture 的 inline JS 使用 ES6+ 语法（const、let、= >、模板字符串），违反 copilot-instructions.md 中的"不使用箭头函数"约定。其他 16 篇文章均遵循 var + function expression 格式。

B-2: microsoft-ai-customer-cases 缺少 2 个共享脚本

该文章未引入 scrollbar.js 和 article-assistant.js。其他 17 篇文章均引入了全部 4 个共享脚本（scrollbar.js、article-presentation.js、article-lightbox.js、article-assistant.js）。

B-3: 文章特有组件未提取到共享 CSS

至少 3 个组件仅在单篇文章 inline style 中定义：timeline-card / era-label（low-code-ai-era）、signal-map / orbit（enterprise-agent-architecture）、quadrant-board（function-calling-landscape）。被其他文章引用时会缺失样式。

B-4: article.css 逼近膨胀预警线

当前 2,828 行，其中演示模式相关规则约占 300+ 行。若继续在 article.css 中追加演示模式、新组件和响应式规则，预计在 5 篇文章后突破 3,500 行。建议评估将演示模式 CSS 拆分为 article-presentation.css。

B-5: 首页 index.html 3,190 行包含全部 JS 逻辑

Galaxy 渲染引擎、Cards 布局、Infographic 布局、主题切换、搜索过滤等全部内联在一个 HTML 文件中。职责边界清晰但文件规模偏大。

  • 组件复用体系成熟 — insight-grid、comparison-grid、flow-list、accordion 等组件在 10+ 篇文章中反复使用，API 设计稳定。新组件（如 timeline-card）可以无缝融入现有体系。
  • 演示模式创新 — 从普通文章到全屏演示的自动化转换（步骤收集、accordion 总览、deck overlay）是差异化能力。PPT 导出的原生对象渲染方案（非截图）保证了可编辑性。
  • 安全合规零缺陷 — 18 篇文章的所有外部链接均通过 rel="noopener noreferrer" 审计。无 XSS 风险（纯静态 HTML，无用户输入处理）。
  • 知识树注册 100% 完整 — 所有 18 篇文章均在 knowledge-data.js 中注册，包含 zh/en 标题和摘要、标签、日期、关联关系。首页自动派生的 planets/groups/posts 数组工作正常。

04 / Action Items

4. 行动项

按优先级排列。C 级项在下次审查前完成；B 级项视开发节奏择机处理。

  1. C1
    提取共享 article-common.js

    将 IntersectionObserver 初始化、accordion toggle 绑定、localStorage 主题/语言同步提取为共享脚本，消除 18 篇文章中的 ~936 行重复代码。

  2. C2
    制定双语内容策略

    明确双语覆盖的最低范围（至少 hero + section 标题 + 导航），或修改 copilot-instructions.md 去掉不切实际的"所有文本双语"要求。

  3. B1
    修复 2 篇文章的 ES6+ 语法

    将 copilot-harness-engineering 和 enterprise-agent-architecture 的 const/let/arrow 改为 var/function expression。

  4. B2
    补充 microsoft-ai-customer-cases 缺失脚本

    添加 scrollbar.js 和 article-assistant.js 引用。

  5. B3
    提取高复用组件到共享 CSS

    将 timeline-card、era-label、verdict-box、attr-card 等组件从 inline style 迁移到 article.css（需同步评估 CSS 膨胀问题）。

  6. B4
    评估 article.css 拆分方案

    将演示模式相关 CSS（~300 行）拆分为 article-presentation.css，控制单文件在 2,500 行以内。

  7. B5
    评估首页 JS 模块化

    将 Galaxy/Cards/Infographic 三种布局引擎拆分为独立 JS 文件，按需加载。当前 3,190 行的单文件对维护构成压力。
