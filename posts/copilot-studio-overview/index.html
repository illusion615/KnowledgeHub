Microsoft Copilot Studio：定位、能力、落地路径与费用

Copilot Studio 是 Microsoft 面向企业的低代码 AI Agent 构建平台，支持通过自然语言描述创建 Agent、连接 1,400+ Power Platform connectors、发布到 Teams / Web / Omnichannel 等渠道。本文从产品定位、核心能力、竞品对比、落地场景、安全治理、快速启动和费用结构七个维度提供选型决策依据。所有数据来源为 Microsoft Learn 官方文档（2026-04-09 核验）。

01 / What Is Copilot Studio

1. 产品定位

Copilot Studio 是 Microsoft Power Platform 的一部分，定位为面向企业的低代码 AI Agent 构建平台。前身为 Power Virtual Agents (PVA)，2023 年更名并整合 generative AI 能力。

01

Agent 定义

Agent 是一个可处理复杂对话、自主决策最佳行动的 AI 伙伴。它协调语言模型、指令、上下文、知识源、Topics、Tools、输入和触发器来完成目标。

02

Agent Flow

Agent Flow 是用于自动化重复性任务的工作流，可由手动触发、其他 Agent 触发或按计划触发。支持自然语言创建或可视化编辑器。

03

两种使用模式

独立 Agent（创建完全自定义的 Agent，完整控制品牌和逻辑）或 M365 Copilot 扩展（用 Agent 扩展 Microsoft 365 Copilot 的能力，使用现有 Copilot 编排器）。

04

运行环境

基于 Dataverse 环境运行。Web App（copilotstudio.microsoft.com）为主要入口；Teams App 仅支持 classic chatbot。GCC / GCC High / DoD 均可用。

02 / Core Capabilities

2. 核心能力

Agent Building

Agent 构建

  • 自然语言创建 — 描述目标（≤1,024 字符），AI 自动生成 name / description / instructions / 建议的 triggers / knowledge / tools。
  • Generative orchestration — Agent 基于 description 语义匹配自动选择 Topics / Tools / Knowledge / 子 Agent，支持多意图链式执行。
  • Classic orchestration — 基于 trigger phrases 匹配选择单个 Topic，适合需要精确控制的场景。
  • Topic 可视化编辑器 — 节点式流程设计：Question / Message / Condition / Set variable / Redirect / Action。
  • Code editor — 支持 YAML 格式的 Topic 代码编辑。
Integration & Extension

集成与扩展

  • Knowledge 来源 — SharePoint / OneDrive / Dataverse / Salesforce / Confluence / ServiceNow / Zendesk / 网页 / 上传文件（DOC / PPT / PDF）。
  • Tools & Connectors — 1,400+ Power Platform connectors（Standard + Premium），支持 HTTP 请求和 custom connectors。
  • Power Automate — 可从 Topic 内调用 Power Automate flows（automated / instant / scheduled）。
  • Bot Framework — 支持 Bot Framework skills 扩展，也可将 Copilot Studio Agent 作为 skill 被其他 Bot 调用。
  • 多语言 — Generative orchestration 自动检测用户语言，支持主语言 + 二级语言配置。

04 / Scenarios & ROI

4. 落地场景与价值度量

以下场景基于 Microsoft 官方文档中的 billing examples 和产品定位描述，配合 Copilot Credits 计费模型给出每日成本估算。

Scenario 1

客户支持 Agent

  • 场景 — 网站客服 Agent，基于退货政策和产品手册回答问题。
  • 交互模式 — 每次会话平均 4 次 classic answer + 2 次 generative answer。
  • 规模 — 900 客户/天。
  • 每日成本 — [(4×1)+(2×2)] × 900 = 7,200 Copilot Credits/天。
  • 价值点 — 客服工单偏转（减少人工客服压力）。
Scenario 2

销售绩效 Agent（M365 Copilot Chat）

  • 场景 — Tenant Graph Grounded Agent，回答员工关于销售数据的问题。
  • 交互模式 — 每次会话 4 次 generative answer + 4 次 tenant graph grounding。
  • 规模 — 50 M365 Copilot 许可用户（免费） + 100 无许可用户。
  • 每日成本 — [(4×2)+(4×10)] × 100 = 4,800 Copilot Credits/天（M365 许可用户不计费）。
Scenario 3

订单处理 Agent（自主触发）

  • 场景 — 收到新订单时自动触发，检查库存、查看物流、审批订单、发送邮件。
  • 交互模式 — 1 次知识检索 + 4 次 agent action。
  • 每次成本 — [(4×5)] = 20 Copilot Credits/次。
  • 价值点 — 端到端流程自动化，无需人工干预。
Evidence Gap

ROI 数据的当前状态

  • Microsoft 客户故事 — Microsoft Customer Stories 网站需登录访问，本文无法直接引用具体企业名称和 ROI 数字。
  • 官方计费示例 — 上述 3 个场景均来自 Microsoft Learn "Billing rates and management" 官方示例，含精确的 Copilot Credits 计算公式。
  • 建议 — 评估 ROI 时使用 Microsoft 官方提供的 Agent Usage Estimator 工具进行定制化估算。

05 / Security & Governance

5. 数据安全与治理

Copilot Studio 遵循 Microsoft Security Development Lifecycle (SDL)，基于商业许可协议（Microsoft Product Terms + Data Protection Addendum）处理数据。

01

DLP 策略

通过 Power Platform admin center 配置数据策略，管控 knowledge sources、actions / connectors / skills、HTTP 请求、发布渠道、AppInsights、触发器等能力的可用性。

02

审计与合规

Maker 审计日志接入 Microsoft Purview。Agent 活动可通过 Microsoft Sentinel 监控和告警。遵循 Microsoft Trust Center 合规标准。

03

加密与数据驻留

支持 Customer-Managed Encryption Keys (CMK)。数据处理位置参见 geographic availability 文档。可禁用跨地理区域的数据移动。

04

访问控制

支持 Entra ID 手动认证（含证书 provider）。用户凭据运行 Tools。Sensitivity Label 在 SharePoint 知识源中显示。环境路由为 Maker 提供隔离的构建空间。

05

发布前安全扫描

Maker 在发布前可看到安全告警（当默认安全配置被修改时）。Agent 运行时保护状态在 Agents 页面可见。

06

Customer Lockbox

支持 Customer Lockbox 安全访问。注意：Lockbox 不覆盖 Agent 365 安全审计日志中发出的数据。

06 / Quick Start

6. 快速启动：从零到发布的 5 步

以下步骤基于 Microsoft Learn "Quickstart: Create and deploy an agent" 官方教程。

1

注册试用

访问 试用注册页面，使用工作/学校邮箱注册。试用期 30 天，可延期 30 天。试用期内可创建和测试 Agent，但无法发布。

2

描述 Agent

在 copilotstudio.microsoft.com 首页输入 Agent 描述（≤1,024 字符），AI 自动生成 name / description / instructions 和建议的 triggers / knowledge / tools。

3

添加 Knowledge

添加知识源：SharePoint 站点 URL、上传 PDF/DOCX/PPTX 文件、或连接 Dataverse 表。Agent 将基于这些知识源生成回答。

4

测试与迭代

在 Test Chat 面板中实时测试。调整 instructions 改变 Agent 语气和行为。修改 Conversation Start 系统 Topic 定制欢迎消息。

5

发布

点击 Publish → 发布到 Demo Website 或 Teams 渠道。正式发布需独立订阅（试用期无法发布）。可设置自定义图标（PNG, <72KB, 192×192px）。

07 / Pricing & Licensing

7. 合作路径与费用结构

Copilot Studio 使用 Copilot Credits 作为计费单位。不同功能消耗不同数量的 Credits。以下数据来源为 Microsoft Learn "Billing rates and management" 文档。

  • 免费试用 — 30 天（可延期 30 天）。可创建和测试 Agent，不可发布。Agent 在过期后继续运行 90 天。
  • 独立订阅 — 通过 Microsoft 365 admin center 购买。获得 Copilot Credits 池，全租户共享。支持所有渠道和 Premium connectors。
  • Teams 计划 — 包含在特定 M365 订阅中。仅限 Teams 渠道，仅 Standard connectors，不支持 generative orchestration、Power Automate flows 和 Bot Framework skills。
  • M365 Copilot 用户 — B2E（面向员工）场景中 Agent 用量包含在 M365 Copilot USL 许可中（Fair Use 限制）。用户需持有 M365 Copilot 许可且 Agent 使用其认证身份运行。
  • Dynamics 365 — Copilot Studio 权限包含在 Dynamics 365 Customer Service 的 Digital Messaging 和 Chat 附加组件中。
  • Pay-as-you-go — 链接 Azure 订阅计费计划。无预付容量约束，超额直接计入 Azure 账单。不受 125% 容量执行策略限制。

单次交互可能同时使用多种功能。例如 Tenant Graph Grounded + Generative Answer = 12 Credits/次。使用 Reasoning 模型时额外加收 premium AI Tools 费率。

  • 触发阈值 — 租户总消耗达到预付容量的 125% 时触发。
  • 执行动作 — 自定义 Agent 被禁用（进行中的会话不中断）。后续调用被拒绝，直到容量增加或重置。
  • 用户看到 — "This agent is currently unavailable. It has reached its usage limit."
  • 通知 — 邮件通知租户管理员 + Power Platform admin center 内通知。
  • 豁免 — Pay-as-you-go 环境不受此策略限制。M365 Copilot 许可用户的 Agent Flow 调用不消耗预付容量。

Microsoft 提供官方 Copilot Studio Agent Usage Estimator，可按 Agent 类型、流量、编排模式、知识源和 Tools 配置生成定制化消耗预估。建议在选型评估阶段使用此工具建模实际场景的 Credits 消耗。
